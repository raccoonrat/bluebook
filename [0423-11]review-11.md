Abstract（最终定稿版）

本蓝皮书讨论的对象，不是一个由 detector、评测组件与治理流程拼接而成的 AI 安全平台，也不是一份控制面架构说明书，而是一套面向企业级 AI 部署的 **保证生产系统（guarantee production system）/义务闭合机器（obligation closure machine）**：它在统一版本面下，把义务语义、运行时判断、强制动作、证据记录与放行陈述闭合为同一可管理对象，从而持续产出 **可部署、可审计、可回放、可收缩的 bounded guarantees**。在这一意义上，控制面不是功能集合，而是企业安全义务得以被生产、维护与辩护的系统边界。

为实现这一对象，本书采用“一主两翼”的结构。**主**为 **Unified AI Security Control Plane**，负责在统一边界内闭合 judgment、execution、evidence 与 release 四个基本对象；**翼 A** 为 **Decoupled Safety**，其作用不是增加一层安全外挂，而是把开放语义空间中不可被完全全局审查的问题，改写为事件可判定、策略可求值、失败可退化、证据可记录的实例级契约；**翼 B** 为 **Evaluation Framework**，其职责也不只是评分，而是把 benchmark constitution、acceptance contract、多轨评测纪律与机制证据包、放行证据包组织成同一证据链，使测量、回归与发布治理进入统一决策。三者的共同目标，不是构建“更大的安全平台”，而是使企业能够在明示前提下，对自身 AI 系统给出合法、可验证、可撤回的安全主张。

本书的核心判断是：企业级 AI 安全的根本问题不是“模型是否更对齐”，而是企业是否拥有一套能够持续闭合 judgment、execution、evidence 与 release 的控制系统。随着企业 AI 从单模型端点演化为包含网关、检索、工具、工作流、记忆与多轮状态的复合系统，风险已扩展为机密性、完整性、可用性、可控性、可审计性、可组合性与效用保持之间的系统性张力。若仍将安全寄托于更强模型、局部护栏或离线分数，义务就会在策略、线上行为、证据链与放行材料之间断裂。控制面的意义，正是在于把这些张力从“各团队局部优化”转化为可版本化 policy、可执行强制、可 join evidence 与可发布 bounded contract。

本蓝皮书给出的是**有界、部署导向的主张**，而不是对开放自然语言交互的全局安全证明。任何安全主张只有在其 scope、preconditions、degradation semantics、deployment regime 与 evidence basis 被明示，并能被编译到 acceptance contract 与 release evidence pack 时，才构成合法的企业级 guarantee statement。相应地，本书将 evaluation 写成 **语义分叉与治理汇合** 的证据引擎：不同法域、语言与义务空间不被压缩为单一总分，而是在共享执行骨架上保持独立语义，并在统一治理决策板上汇合为 risk profile、release recommendation 与 audit artifact。最终，本书提出的不是一组能力清单，而是一套可运行的 enterprise operating model：它把 theory、system、benchmark 与 release 压成同一制品链路，并使每一次发布都成为一份可辩护、可回放、可回滚的 bounded contract。

Executive Summary（最终定稿版）

企业级 AI 安全的核心，不再是“如何为模型增加更多安全能力”，而是“如何持续生产、维护并辩护可签署的安全保证”。本蓝皮书因此提出的总对象，不是一个 AI 安全平台，也不是一套控制面模块，而是一台 **保证生产系统**：它要求企业在统一版本面下，将义务语义、运行时判断、强制动作、证据记录与 release 决策闭合为同一对象，使安全主张能够被生产、验证、收缩、回放并进入发布治理。Unified AI Security Control Plane 是这一对象在系统层的边界形式，而不是对它的全部解释。

这一总对象以“一主两翼”的方式被组织起来。主轴是 **Unified AI Security Control Plane**，其任务是在统一边界内闭合 judgment、execution、evidence 与 release 四个核心对象；第一翼是 **Decoupled Safety**，它将安全从模型内部难以稳定引用的隐式属性，改写为 policy、event、decision、evidence 与 Fail-Safe 对象上的实例级契约；第二翼是 **Evaluation Framework**，它把 benchmark constitution、acceptance contract、多轨评测纪律与两类 evidence pack 组织为同一证据引擎，使系统的测量、回归与放行答辩进入统一治理语法。由此，AI 安全不再被理解为“局部防御能力的叠加”，而被理解为一种能够持续产出 bounded guarantees 的企业系统。

本书之所以必须采用这一视角，是因为企业级 AI 部署已不再是单模型补全问题，而是一个由网关、检索、工具、工作流、记忆、多轮状态与审计流程共同构成的复合系统。风险也因此不再局限于有害输出，而扩展为机密性、完整性、可用性、可控性、可审计性、可组合性与效用保持之间的系统性张力。若仍将安全寄托于更大的模型、更多的离线对齐，或若干松耦合护栏的局部最优，组织将无法在 policy、线上行为、证据链与放行材料之间维持一致性。安全义务最终会断裂为孤立分数、孤立日志与孤立拦截，而无法成为可签署、可治理的 release object。

在系统原理上，本书提出的不是“再增加一层安全外挂”，而是一条必要路径：面对开放语义空间中无法被完全在线全局审查的问题，系统必须退到一个 **可计算外壳（computable envelope）** 上。Decoupled Safety 的意义，正在于把开放任务空间压缩为有限的 event、有限的 decision 语义、有限的 Fail-Safe 动作空间与有限的 evidence 字段，使热路径内的决策可以被确定性求值、被写入证据、并在冲突时进入协议化收缩。由此，运行时不再依赖模型在压力情境下的自我反思，而是由 policy、kernel、FailSafeObject 与 evidence graph 共同承担可执行的安全义务。

在方法论上，本书将企业级 AI 安全建模为一个 **双时间尺度的风险自适应控制系统**。快时间尺度位于在线热路径：系统在单请求、单会话预算内完成 event 求值、动作派发、Fail-Safe 收敛与证据追加；慢时间尺度位于离线冷路径：系统通过 benchmark、risk profile、multi-track evaluation、release gating 与 regression 持续更新 policy_version、检测器包、能力包与放行边界。Evaluation 在此不是评分器，而是 **decision-signal compiler**；Evidence 与 Release Gating 也不是末端归档，而是把判断与执行提升为可放行、可审计、可回归的保证对象。

在评测与治理层，本书拒绝把不同风险语义压缩为单一总分，而采用 **semantic bifurcation with governance convergence** 的结构。全球安全稳健性、法域合规性、语言特定风险与组织义务在共享执行骨架上并行存在，各自保留 suite、aggregation logic 与 report semantics；它们最终汇入统一治理决策板，输出的是放行、限权放行、灰度或回滚等 release decision，而不是被隐式压缩为一个分数。因此，Track A 与 Track B 的纪律是本书的硬约束：Track A 承载可部署结论，Track B 承载机制上界与压力验证，二者不得被混读或混算。所有对外与对内可陈述保证，都必须由 Mechanism Evidence Pack 与 Release Evidence Pack 支撑。

本书同时明确给出一套 **claim discipline / guarantee grammar**。任何允许进入 release defense、对外材料或合同文本的安全主张，必须同时声明其 scope、invariants、degradation semantics、preconditions、evidence basis 与 deployment regime；任何无法被 acceptance contract 与 evidence pack 编译的陈述，都不被视为合法 guarantee statement。由此，本书提供的不是全局自然语言安全证明，而是一套 bounded deployment contract：在明示前提成立时，系统可以主张哪些有界保证；当前提破坏时，系统又必须退入何种最低服务形态，并同步收缩对外主张。

最后，本蓝皮书并不把 unified R&D framework 理解为组织汇报结构，而把它理解为一个 operating program。通过 **theory → system → benchmark → release** 的统一制品链路，以及 **One DTL Security Program** 的单一真源对象、单一放行门与单一演化节律，企业可以把语义与义务生成、运行时控制实现、测量与证据编译、以及放行与监督闭环，组织为同一条保证生产线。真正的护城河，不在于拥有更多模块，而在于拥有一套能够持续把风险语义编译为 control、把 control 沉淀为 evidence、再把 evidence 编译为 bounded guarantees 的企业系统。
